Migrating to a Cloud-Native Microsoft Fabric Platform

Leading the transformation of the data function from manual Excel, CSV and Access-based processes to a governed Microsoft Fabric lakehouse with a Bronze–Silver–Gold medallion architecture — cutting manual processing time by roughly 80%.

Context

The data function had grown organically around spreadsheets and an Access database. It worked — but it didn't scale. Every monthly cycle meant manually pulling files together, re-keying and reconciling, with limited auditability and a single point of failure in the people who knew where everything lived.

I led the move to a cloud-native platform on Microsoft Fabric, designing the architecture and building the pipelines that now underpin all reporting for the portfolio.

The challenge

  • Manual, fragile processes — Excel/CSV and Access workflows that were slow, error-prone and hard to audit.
  • No clear lineage — it was difficult to trace a number on a report back to its source.
  • Scaling pains — a growing portfolio and more data sources made the manual approach unsustainable.
  • Environments & safety — changes were made directly against live data, with no real separation of dev, test and production.

The solution: a medallion lakehouse on Fabric

I designed and implemented a Microsoft Fabric platform built around a medallion architecture, with automated pipelines moving data through clearly defined layers.

  1. Bronze — raw landing. Source files and feeds (income, budgets, lettings, arrears, expenditure, stock, compliance, complaints) land in OneLake exactly as received, giving an immutable, auditable record.
  2. Silver — cleansed & conformed. Data Factory pipelines validate, cleanse, deduplicate and standardise keys so multi-source data lines up consistently.
  3. Gold — curated & analytics-ready. Business-ready tables modelled for reporting feed the Power BI semantic model as a single, trusted source.

Dev / test / prod. The lakehouse, warehouse, Data Factory and real-time analytics run across separate environments, so changes are built and validated safely before they ever touch production reporting.

Automation & orchestration

Pipelines are scheduled and orchestrated in Data Factory, with validation and cleansing built in rather than bolted on. The result is a repeatable, scalable refresh that produces auditable, analytics-ready datasets every cycle — replacing the manual assembly that used to dominate the month.
