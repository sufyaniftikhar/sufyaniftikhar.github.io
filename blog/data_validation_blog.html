A Dictionary-Driven Data Validation Framework

A Python solution that uses a centralised, human-readable Excel data dictionary to automate data-quality checks and generate actionable error reports — letting non-technical users own the rules.

Type
Data Governance & Automation
Stack
Python · Pandas · JSON · Excel
Impact
Streamlined quality reporting

The challenge

Maintaining data quality is a constant struggle. When new data is ingested it must be validated against business rules to ensure integrity — yet this is often manual, with data managers visually inspecting files. That doesn't scale and is highly error-prone.

The goal was a flexible, automated validation process that non-technical users could manage — a central place to define rules (a data dictionary) and a script to apply them to incoming data.

The solution: a dictionary-driven script

I built a Python solution that uses a simple Excel file as a data dictionary to drive the validation logic — so data managers can define and update rules without touching code.

  1. The Excel data dictionary — each row defines the rules for a field: data type, min/max, and allowable values or patterns.
  2. Conversion to JSON — the dictionary is converted to machine-readable JSON for the script to parse.
  3. Automated Python validation — the script reads the rules and the data, applying each rule to every field.
  4. Actionable error reporting — it generates an Excel report flagging issues, with valid and failed rows on separate sheets.

2 · Converted to JSON

{
    "EmployeeID": { "data_type": "integer", "min_value": 1000, "max_value": 9999 },
    "Department": { "data_type": "string", "allowable_values": ["Sales", "Marketing", "IT", "HR"] },
    "Salary": { "data_type": "integer", "min_value": 30000, "max_value": 150000 }
}

3 · The Python validation script

import pandas as pd
import json

def validate_row(row, rules):
    """Applies a set of rules to a single row of data."""
    errors = []
    for column, ruleset in rules.items():
        value = row[column]

        if 'allowable_values' in ruleset and value not in ruleset['allowable_values']:
            errors.append(f"'{value}' is not an allowable value for {column}.")

        if 'min_value' in ruleset and value < ruleset['min_value']:
            errors.append(f"{column} value {value} is below minimum of {ruleset['min_value']}.")

    return errors if errors else None

# --- Main script ---
with open('data_dictionary.json') as f:
    validation_rules = json.load(f)
data_df = pd.read_excel('employee_data.xlsx')

error_log, valid_rows = [], []
for index, row in data_df.iterrows():
    row_errors = validate_row(row, validation_rules)
    if row_errors:
        for error in row_errors:
            error_log.append({'Row': index + 2, 'Error': error})
    else:
        valid_rows.append(index)

with pd.ExcelWriter('validation_report.xlsx') as writer:
    pd.DataFrame(error_log).to_excel(writer, sheet_name='Errors', index=False)
    data_df.iloc[valid_rows].to_excel(writer, sheet_name='Valid Data', index=False)

  • Empowers non-technical users — data managers control validation rules from a simple Excel file.
  • Improves data quality — all incoming data is systematically checked.
  • Increases efficiency — automates a previously manual, time-consuming task.
  • Clear, actionable reporting — error reports make issues easy to pinpoint and fix.
